# Build Prompt — v0.4c (cleanup): don't write per-day report files in range/tournament mode

**For the coding session.** Read `CLAUDE.md` first. Tiny, surgical cleanup. Do exactly this.

## The problem

`run_one_day` (in `same_day_lab/runner.py`) writes a per-day JSON+MD report for every day.
`run_range` and `run_tournament` call it per day (and the tournament does it per strategy ×
per window), so a single `run-range` over ~10 days, or a `tournament` over 3 strategies × 2
windows, litters `reports/` with dozens of incidental per-day files. The aggregate /
tournament report is the deliverable; the per-day detail is already in the aggregate's
per-day table and in the per-day `runs`/`trades` DB rows. The per-day files are clutter.

## The fix

Let `run_one_day` skip the per-day **file** write when it's invoked as part of a rollup,
while the single-day `run` command keeps writing its file (unchanged).

- Thread a flag through `run_one_day` — e.g. `write_report_file: bool = True` (or treat
  `reports_dir=None` as "don't write the per-day files"). `run_range` and `run_tournament`
  pass the no-write option; the `run` CLI command passes the writing option (default).
- **Still compute the report dict** (the aggregate/tournament rollups + determinism depend
  on it) and **still persist the per-day `runs`/`trades` DB rows** — only the per-day
  JSON/MD *file writes* are suppressed in rollup mode.
- **No dangling paths:** when no per-day file is written, the `runs` row's
  `report_json_path`/`report_md_path` for that day must be empty/null (not a path to a
  file that doesn't exist). The aggregate/tournament report file remains the durable
  artifact and is still written.

## Invariants to preserve
- Single-day `run` is **unchanged** (still writes its JSON+MD, same `report_hash`).
- `report_hash`, `aggregate_hash`, `tournament_hash` are **unchanged** (this only affects
  whether incidental per-day files land on disk, not any analytical content).
- Determinism and all existing behavior otherwise identical.

## Do NOT
- Touch fill/verdict/strategy/aggregate/tournament analytical logic.
- Remove the per-day `runs`/`trades` DB rows (keep them; just null their file paths in
  rollup mode), or change the aggregate/tournament report shape.
- Edit `README.md` / `CLAUDE.md` — report any doc-impacting note back to the planning agent.
